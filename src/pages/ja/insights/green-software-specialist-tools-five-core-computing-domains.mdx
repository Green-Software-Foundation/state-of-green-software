専門グリーンソフトウェアツールが対応するコンピューティングには、5つのコアドメインがあり、それぞれ成熟度が異なります。コアドメインは、[AI](https://www.techtarget.com/searchenterpriseai/feature/Green-AI-tackles-effects-of-AI-ML-on-climate-change)、[コード](https://www.ibm.com/cloud/blog/green-coding)、[オペレーティングシステム](https://www.redalyc.org/pdf/2652/265219635005.pdf)、[クラウド](https://www.accenture.com/us-en/insights/strategy/green-behind-cloud)、および[ウェブ](https://www.thegreenwebfoundation.org/)です。
![Five core computing domains](/ja/images/insights/green-software-specialist-tools-five-core-computing-domains-additional-1_J.png)

1. AI:AIグリーンソフトウェアのツールはまだ初期段階です。成熟したツールもありますが、大半のユーティリティは学術論文の概念実証として存在し、広く普及していません。発見容易性（ひいては普及性）は依然として限定的です。

2. コード：最近、ソフトウェアアプリケーションのエネルギーや炭素強度をコードレベルで測定し、削減するツールが登場しています。このようなユーティリティはまだ新しく、数も少ないですが、よりきめ細かい指標や微調整に関して大きな可能性を秘めています。

3. OS:Windows、Linux、OSX、およびAndroidなどのOSは、省エネモードから炭素を考慮したアップデートまで、グリーンソフトウェアツールを搭載することが多くなっています。また、オープンソースおよび商用を問わず、グリーンソフトウェアユーティリティのエコシステムもコミュニティによって開発されています。

4. クラウド：クラウドプロバイダー向けのグリーンソフトウェアツールは、ウェブ向けよりも狭い範囲ですが、はるかに大きな規模で、莫大な影響力をもって運用されています。クラウドプロバイダーは、環境を考慮したツールに多額の投資を行っており、これらは、独立したオープンソースや商用のグリーンクラウド製品によって補完されています。

5. ウェブ：ウェブ上ではツールが急速に増え、GitHubや学術文献でも数多く発見されています。ほとんどの場合、ウェブに特化したツールは、クラウドに特化したツールのような規模でなく、リソースも備えませんが、より広範な製品やサービスを包含しています。

これらのドメインは相互に排他的でなく、ソフトウェア実践者は、これらを組み合わせて使用し、炭素排出量について最も正確な結果と最適な影響を達成する必要があります。
