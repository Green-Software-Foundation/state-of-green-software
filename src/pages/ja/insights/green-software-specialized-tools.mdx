GitHubプラットフォームで公開されているグリーンソフトウェアツールのうち、名前、トピック、または説明文のいずれかに環境的側面が明示されているものを専門グリーンソフトウェアツールと定義すると、約800～1,200のグリーンソフトウェアツールが公開されています。GitHubのリポジトリにあるツールと、学術的なインデックスや一般的なウェブ検索で見つかるツールを合わせると、現在2,000の専門グリーンソフトウェアツールが存在すると、Green Software Foundation（GSF）は推定しています。

GitHubの公開リポジトリの基準ラインを800～1,200とする上で、GSFは以下のステップをたどりました：

1. [Green Software Foundation](https://github.com/Green-Software-Foundation/awesome-green-software)、[The Green Web Foundation](https://github.com/thegreenwebfoundation/)、[Open Sustainable Technologies](https://github.com/protontypes/open-sustainable-technology)、[Collectif Conception Numérique Responsable](https://github.com/cnumr)、および[Open Sustainable Technologies](https://github.com/protontypes/open-sustainable-technology)など、各分野の専門家が整理したリストから、よく知られた専門グリーンソフトウェアツールを特定しました。
2. 整理された専門グリーンソフトウェアツールで、タグ付けされたGitHubの全トピック（キーワード）をリストアップし、それぞれに対して検索をかけ、検索結果の全キーワードを集計しました。このとき、「サステナビリティ」のようにキーワードが広すぎる場合、関連性のない結果を除外するためのデータクリーニングが必要でした。
3. 関連するすべてのリポジトリで単語頻度分析を作成しました。
![2000 green software tools](/images/insights/green-software-specialized-tools-additional-1.png)
4. 上記のグリーンソフトウェアのキーワードについて、最も使用頻度が高いものを検索したところ、約800件の検索結果が得られました。

その総数は、境界線をどこに引くかによって異なります。あるリポジトリは[単なるフォーク](https://github.com/thegreenwebfoundation/gitpod-node-starter)（クローン）や[使えない未完成状態](https://github.com/228298/BigData)であり、またあるリポジトリは、[学術論文のワンオフコード](https://github.com/Jingjing-NLP/KNAS)です。Green Software Foundationは、より包括的な境界線を見積り、つまり、より不明瞭なキーワードを追加し、ツールとして奨励されていないワンオフコードを含めることで、1,200の上限で、最大50%以上のヒット数を追加できると考えます。どのような境界線を設定するにしても、環境への配慮や目標について明示的に言及するツールを専門ソフトウェアツールと定義する場合、上限はせいぜい数千になることは明らかです。

GitHub以外にも、[Green Spector](https://greenspector.com/en/home/)など、ソフトウェアによる環境への影響に明確に注目したツールや、グーグル、アマゾン、マイクロソフトなどのクラウドプロバイダーが商業ベースで提供しているものがあります。

同様に、ソフトウェアによる環境への影響に焦点を当てた[学術文献に記載されているツール](https://scholar.google.co.uk/scholar?hl=en&as_sdt=0%2C5&q=%22green+software%22+intitle%3Atool&btnG=)は、再利用可能なツールとしてではなく、特定の研究プロジェクトの概念実証や実証結果を達成するために設計された独自のものです。学術文献ではこれらのツールについて言及していますが、孤立した学術的研究を超えて、これらのツールが普及したことを示すものはありません。現在までに、Green Software Foundationは、600〜1,000の同様ツールを特定しています。

2022年だけでも8,600万もの新しいGitHub[リポジトリ](https://octoverse.github.com/2022/developer-community)がありますが、グリーンソフトウェアツールのエコシステムは、より大きなソフトウェアエコシステムと比較してまだ初期段階にあるため、提案された2,000という数字を調整しても大きな差異はないでしょう。
